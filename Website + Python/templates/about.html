Intentions of the Project

The intention of this project is to allow a user to indentify whether their device or network is suseceptable to Shodans port scanning methodology, and then even further,
to allow a user to check whether their device or network is listed in shodans database, and whether it is listed as vulnerable or not.



First off, the page uses a similar technique to Shodan, scanning to check if a specific port is open, or not.
The port for MQTT devices is "1883", so the program can check specifically to see whether this port is open or not, if it is, the program will respond accordingly.


The main functionality of this program is to use Shodan's API and filters function to take a query the user inputs (ip address) and find
out two things:

-Does a given device or webpage have an open MQTT port?


-Is that device listed in Shodan's database
If it is, would it be considered vulnerable by shodan's OSINT system?



What is MQTT?

Message Queuing Telemetry Transport is a publish-subscribe networking protocol, this protocol can be used over any protocol that provides
ordered, lossless, bi-directional connections, but is traditionally used over transmission control protocol (TCP) or internet protocol-based networks.

It provides a low processing power solution to long-distance IoT device communicaiton; most networks include a broker, a middle-man that can sort and send
"topics" that are needed by IoT devices to other IoT devices.

These topics are essentially a set of rules or tables that order pieces of data, these are sent intially by IoT devices as "submit packets"

Once the target IoT device recieves a "submit packet", it can fill out this data, and send it back the same route through a "publishing packet".
